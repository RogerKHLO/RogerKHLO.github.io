Telegram Reservation Bot

"Order from chaos in the kitchen."
Technical Specs
Client -
Stack GAS, REST, Sheets
Cost $0 (Serverless)
Outcome Automated Booking

The Brief

Small food and beverage businesses often drown in paper logs and inconsistent messaging apps. The objective was to eliminate the friction of manual booking management without incurring the cost of enterprise SaaS solutions.

The target system needed to be lightweight, mobile-first, and capable of handling "fuzzy" human inputs (e.g., "Table for 4 next Friday") while maintaining a rigorous, structured database backend.

The Arsenal

The solution utilizes a serverless architecture, leveraging Google's ecosystem to act as both the backend logic and the database.

Google Apps Script
The Engine. Hosts the `doPost(e)` webhook to parse incoming JSON payloads from Telegram securely.
Telegram Bot API
The Interface. Provides a secure, mobile-friendly command line for staff to manage bookings on the fly.
Google Sheets
The Vault. Acts as a real-time relational database, automatically organizing data by month.
Date Parser
The Logic. Custom regex algorithms to interpret natural language inputs (e.g., "21/7", "Tomorrow").

The Roadmap

While the current system is operational, several upgrades are within consideration to expand capabilities:

  • Dialogflow Integration: Moving beyond Regex to full Natural Language Processing (NLP) for complex intent matching and conversational context.
  • Voice Support (Twilio): Implementing an automated voice agent to handle reservations via phone calls for non-digital customers.
  • Public Web Portal: Developing a customer-facing frontend to allow direct bookings, supplementing the internal staff-only Telegram interface.
  • Analytics Dashboard: A visualization layer to track peak booking times, cancellation rates, and monthly covers directly from the Google Sheets data.

The Execution

The bot operates via a secure HTTPS webhook. When a staff member inputs a command like /cancel +60123456789 21/7, the script intercepts the payload, sanitizes the phone number, locates the record in the cloud database, and removes it—instantly confirming the action back to the chat.

The Result

Deployment resulted in a centralized, zero-cost reservation system. It eliminated double-bookings caused by paper log errors and reduced staff training time to minutes due to the intuitive chat interface.
